Core OOP Concepts

P01 – P04
P01 Class and Object Core OOP · 7 marks
A class is a blueprint or template that defines the structure (attributes) and behaviour (methods) of objects. An object is a real-world entity — a live instance created from that class using the new keyword. Each object gets its own copy of the class's instance variables.
class = blueprint object = instance new keyword creates objects each object has own variable copies
Java
class Student { String name; // attribute (instance variable) int age; // attribute void display() { // method System.out.println("Name: " + name + ", Age: " + age); } } public class Main { public static void main(String[] args) { Student s1 = new Student(); // object creation s1.name = "Alice"; s1.age = 20; s1.display(); // Output: Name: Alice, Age: 20 Student s2 = new Student(); // another object s2.name = "Bob"; s2.age = 22; s2.display(); // Output: Name: Bob, Age: 22 } }
Output Name: Alice, Age: 20 Name: Bob, Age: 22
Key: s1 and s2 are two separate objects — changes to s1.name do NOT affect s2.name. Both share the same class definition but hold their own data.
P02 Method Overloading Polymorphism · 7 marks
Method Overloading means having multiple methods with the same name but different parameter lists (different number or types of parameters) within the same class. Java selects the correct method at compile time — this is called static polymorphism or compile-time polymorphism.
same method name different parameters resolved at compile time static polymorphism
Java
class Calculator { // overload 1: two ints int add(int a, int b) { return a + b; } // overload 2: two doubles double add(double a, double b) { return a + b; } // overload 3: three ints int add(int a, int b, int c) { return a + b + c; } } public class Main { public static void main(String[] args) { Calculator calc = new Calculator(); System.out.println(calc.add(3, 4)); // calls overload 1 → 7 System.out.println(calc.add(2.5, 3.5)); // calls overload 2 → 6.0 System.out.println(calc.add(1, 2, 3)); // calls overload 3 → 6 } }
Output 7 6.0 6
Rule: Return type alone CANNOT distinguish overloaded methods. The parameter list (number, type, or order) must differ.
P03 Method Overriding Polymorphism · 7 marks
Method Overriding is when a subclass provides its own implementation of a method already defined in its superclass. The method name, parameters, and return type must be exactly the same. The correct version is selected at runtime — this is dynamic polymorphism.
same signature as parent child redefines behaviour resolved at runtime dynamic polymorphism
Java
class Animal { void sound() { System.out.println("Some generic sound"); } } class Dog extends Animal { void sound() { // overrides Animal.sound() System.out.println("Bark!"); } } class Cat extends Animal { void sound() { // overrides Animal.sound() System.out.println("Meow!"); } } public class Main { public static void main(String[] args) { Animal a = new Dog(); // parent reference, child object a.sound(); // Output: Bark! (runtime decides) a = new Cat(); a.sound(); // Output: Meow! } }
Output Bark! Meow!
Overloading vs Overriding: Overloading = same class, different params, compile time. Overriding = parent-child, same params, runtime.
P04 Access Modifiers (Visibility) Encapsulation · 7 marks
Java has 4 access modifiers that control where a member (variable/method) can be accessed from. private is most restrictive (same class only); public is least restrictive (accessible everywhere).
private → same class only default → same package protected → package + subclasses public → everywhere
Java
class Example { private int x = 10; // accessible only inside this class int y = 20; // default (friendly) — same package only protected int z = 30; // same package + subclasses public int w = 40; // accessible from anywhere public void show() { System.out.println("x=" + x + " y=" + y + " z=" + z + " w=" + w); } } public class Main { public static void main(String[] args) { Example e = new Example(); e.show(); // OK — show() is public // e.x = 5; // ERROR — x is private System.out.println(e.w); // OK — w is public → 40 } }
Output x=10 y=20 z=30 w=40 40
Best practice: Declare variables as private and provide public getters/setters — this is encapsulation.

P05 Types of Inheritance Inheritance · 7 marks
Inheritance lets a child class acquire properties and behaviours of a parent class using the extends keyword. Java supports Single, Multi-level, and Hierarchical inheritance directly. Multiple and Hybrid are achieved through interfaces.
Single: Animal ← Dog Hierarchical: Animal ← Dog ← Cat Multi-level: Animal ← Dog ← Puppy Multiple: (via interfaces only)
Java
// ── 1. Single Inheritance ── class Animal { void eat() { System.out.println("eating"); } } class Dog extends Animal { } // Dog inherits eat() // ── 2. Hierarchical Inheritance ── class Cat extends Animal { } // Cat also inherits eat() // ── 3. Multi-Level Inheritance ── class Puppy extends Dog { } // Puppy → Dog → Animal (chain) // ── 4. Multiple Inheritance (via interface) ── interface Swimmer { void swim(); } interface Runner { void run(); } class Labrador extends Dog implements Swimmer, Runner { public void swim() { System.out.println("Swimming"); } public void run() { System.out.println("Running"); } } public class Main { public static void main(String[] args) { Labrador lab = new Labrador(); lab.eat(); // inherited from Animal lab.swim(); // from Swimmer lab.run(); // from Runner } }
Output eating Swimming Running
P06 Multilevel Inheritance — Student → Marks → Result Inheritance · 7 marks
Multilevel inheritance creates a chain: Student → Marks → Result. Each class extends the one above it. The bottom class (Result) inherits all members from both Marks and Student.
A extends B extends C bottom class inherits everything NOT the same as multiple inheritance
Student (name, rollNo) ↓ extends Marks (marks1, marks2) ↓ extends Result (total, display())
Java
class Student { String name; int rollNo; void getStudent(String n, int r) { name = n; rollNo = r; } } class Marks extends Student { int marks1, marks2; void getMarks(int m1, int m2) { marks1 = m1; marks2 = m2; } } class Result extends Marks { int total; void display() { total = marks1 + marks2; System.out.println("Name : " + name); System.out.println("Roll No: " + rollNo); System.out.println("Marks : " + marks1 + " + " + marks2); System.out.println("Total : " + total); } } public class Main { public static void main(String[] args) { Result r = new Result(); r.getStudent("Alice", 101); r.getMarks(85, 90); r.display(); } }
Output Name : Alice Roll No: 101 Marks : 85 + 90 Total : 175
P07 Hierarchical Inheritance — TwoWheeler → MotorBike & Cycle Inheritance · 7 marks
Hierarchical inheritance: one parent class, two child classes. Both MotorBike and Cycle extend TwoWheeler and inherit its wheels variable and showWheels() method.
TwoWheeler (wheels=2, showWheels()) / \ MotorBike Cycle (brand, engineCC) (type)
Java
class TwoWheeler { int wheels = 2; void showWheels() { System.out.println("Wheels: " + wheels); } } class MotorBike extends TwoWheeler { String brand; int engineCC; public MotorBike(String b, int cc) { brand = b; engineCC = cc; } void display() { showWheels(); // inherited method System.out.println("Brand : " + brand); System.out.println("Engine: " + engineCC + "cc"); } } class Cycle extends TwoWheeler { String type; public Cycle(String t) { type = t; } void display() { showWheels(); // inherited method System.out.println("Type : " + type); System.out.println("No engine (pedal-powered)"); } } public class Main { public static void main(String[] args) { MotorBike mb = new MotorBike("Honda", 150); mb.display(); System.out.println("---"); Cycle cy = new Cycle("Mountain Bike"); cy.display(); } }
Output Wheels: 2 Brand : Honda Engine: 150cc --- Wheels: 2 Type : Mountain Bike No engine (pedal-powered)
P08 Person → Student (Inheritance + Subclass) 2024 Paper Q VII · 7 marks
Student extends Person. The child class adds a grade field and overrides printDetails() to show all three attributes. super(name, age) forwards the first two parameters to the parent constructor.

P09 super Keyword — Three Uses 2023 Paper Q VIII · 7 marks
The super keyword references the immediate parent class. It has 3 uses: (1) call parent constructor, (2) call parent method, (3) access parent variable when child shadows it with same name.
super() → parent constructor super.method() → parent method super.field → parent variable super() must be first statement
Java
class Person { private String name; public Person(String name) { this.name = name; } public void introduce() { System.out.println("Hi, my name is " + name); } } class Student extends Person { private int gradeLevel; public Student(String name, int gradeLevel) { super(name); // use 1: call parent constructor this.gradeLevel = gradeLevel; } public void study() { super.introduce(); // use 2: call parent method System.out.println("I'm in grade " + gradeLevel); } } public class Main { public static void main(String[] args) { Student s = new Student("John", 9); s.study(); } }
Output Hi, my name is John I'm in grade 9

Abstract Classes & Interfaces

P10 – P12
P10 Abstract Class and Abstract Method Teacher's list · 7 marks
An abstract class cannot be instantiated (no objects directly). It may contain abstract methods (no body — just a signature) that all subclasses must implement. It can also have normal (concrete) methods with bodies.
abstract keyword cannot use new AbstractClass() forces subclasses to implement can have concrete methods too
Java
abstract class Animal { String name; abstract void makeSound(); // abstract — no body, must override void breathe() { // concrete method — has body System.out.println("Breathing..."); } } class Dog extends Animal { void makeSound() { // must implement abstract method System.out.println("Bark!"); } } class Cat extends Animal { void makeSound() { System.out.println("Meow!"); } } public class Main { public static void main(String[] args) { // Animal a = new Animal(); // ERROR — cannot instantiate Dog d = new Dog(); d.makeSound(); // Output: Bark! d.breathe(); // Output: Breathing... new Cat().makeSound(); // Output: Meow! } }
Output Bark! Breathing... Meow!
P11 Interface — Multiple Inheritance 2023/2024 Papers · 7 marks
An interface contains only abstract methods (and constants). A class can implement multiple interfaces — this achieves multiple inheritance in Java. All interface methods are public abstract by default; all variables are public static final.
interface keyword implements (not extends) class can implement many interfaces all methods implicitly abstract
Java
interface Animal { void makeSound(); // public abstract by default void eat(); } interface Pet { void play(); } class Dog implements Animal, Pet { // implements BOTH interfaces public void makeSound() { System.out.println("Woof!"); } public void eat() { System.out.println("Eating dog food"); } public void play() { System.out.println("Playing fetch"); } } public class Main { public static void main(String[] args) { Dog d = new Dog(); d.makeSound(); // Woof! d.eat(); // Eating dog food d.play(); // Playing fetch } }
Output Woof! Eating dog food Playing fetch
Interface vs Abstract Class: Use interface for "can-do" contracts (Swimmable, Runnable). Use abstract class for shared "is-a" base structure.
P12 Abstract Class vs Interface — Examples Teacher's list · 7 marks
Two side-by-side examples showing the key difference: abstract class Shape has instance variables and concrete methods; interface Drawable only has method contracts.
Java — Abstract Class example
// ── Abstract Class ── abstract class Shape { String color; // instance variable (allowed) abstract double area(); // abstract method void display() { // concrete method (allowed) System.out.println("Color: " + color); } } class Circle extends Shape { double radius; double area() { return 3.14 * radius * radius; } } // ── Interface ── interface Drawable { void draw(); // public abstract by default } interface Resizable { void resize(); } class Rectangle implements Drawable, Resizable { public void draw() { System.out.println("Drawing rectangle"); } public void resize() { System.out.println("Resizing rectangle"); } } public class Main { public static void main(String[] args) { Circle c = new Circle(); c.color = "Red"; c.radius = 5; c.display(); // Color: Red System.out.println("Area: " + c.area()); // 78.5 Rectangle r = new Rectangle(); r.draw(); // Drawing rectangle r.resize(); // Resizing rectangle } }
Output Color: Red Area: 78.5 Drawing rectangle Resizing rectangle

P13 final Variable, final Method, final Class Teacher's list · 7 marks
The final keyword has 3 uses: (1) final variable = constant, cannot reassign; (2) final method = cannot be overridden; (3) final class = cannot be extended/inherited.
final var → constant final method → no override final class → no subclass String is a real final class
Java
// ── 1. Final Variable ── final int MAX_SIZE = 100; // MAX_SIZE = 200; // COMPILE ERROR — cannot reassign // ── 2. Final Method ── class Parent { final void display() { System.out.println("This is a final method"); } } class Child extends Parent { // void display() { } // COMPILE ERROR — cannot override final } // ── 3. Final Class ── final public class MyFinalClass { void show() { System.out.println("Final class!"); } } // class Sub extends MyFinalClass { } // COMPILE ERROR public class Main { public static void main(String[] args) { System.out.println("MAX_SIZE = " + MAX_SIZE); new Parent().display(); new MyFinalClass().show(); } }
Output MAX_SIZE = 100 This is a final method Final class!
Real examples: java.lang.String and java.lang.Integer are final classes — that's why you can't extend String.
P14 Static Variables and Static Methods 2023 Paper Q VI · 7 marks
Static members belong to the class, not to any individual object. Static variables are shared across all instances. Static methods can be called without creating an object — using just the class name.
one copy for all objects accessed via ClassName.member no this/super inside static methods main() is always static
Java
// ── Static Variable ── class Counter { static int count = 0; // ONE copy shared by all objects Counter() { count++; } // increments shared count } // ── Static Method ── class MathHelper { static int square(int n) { return n * n; } static int cube(int n) { return n * n * n; } } public class Main { public static void main(String[] args) { new Counter(); new Counter(); new Counter(); System.out.println("Objects created: " + Counter.count); // 3 // No object needed — call directly on class System.out.println("square(5) = " + MathHelper.square(5)); // 25 System.out.println("cube(3) = " + MathHelper.cube(3)); // 27 } }
Output Objects created: 3 square(5) = 25 cube(3) = 27
P15 Exception Handling (try / catch / finally) 2023 Paper Q V · 7 marks
An exception is a runtime error that disrupts normal program flow. Java uses a try-catch block to handle exceptions gracefully. The finally block always executes — useful for cleanup (closing files, connections).
try → risky code catch → handle the error finally → always runs throw / throws → propagate
Java
class ExceptionHandling { public static void main(String args[]) { int a[] = {5, 10}; int b = 5; try { int x = a[2] / b - a[1]; // a[2] doesn't exist! } catch (ArithmeticException e) { System.out.println("Caught: Division by zero"); } catch (ArrayIndexOutOfBoundsException e) { System.out.println("Caught: Array index out of bounds"); } finally { System.out.println("finally block always executes"); } // program continues normally after catch int y = a[1] / a[0]; System.out.println("y = " + y); // Output: y = 2 } }
Output Caught: Array index out of bounds finally block always executes y = 2
Execution flow: try → exception thrown → matching catch executes → finally always runs → program continues after try-catch block.

P17 3 Ways to Access a Package 2023 Paper Q II.3 · 3 marks
There are exactly 3 ways to access a package from outside. Each has different granularity — from importing everything, to importing one class, to writing the full class path each time.
Java
// Way 1: import all classes in the package (wildcard) import java.util.*; Scanner sc1 = new Scanner(System.in); // Way 2: import a specific class only import java.util.Scanner; Scanner sc2 = new Scanner(System.in); // Way 3: use the fully qualified name (no import needed) java.util.Scanner sc3 = new java.util.Scanner(System.in);
Best practice: Prefer Way 2 (specific import) — it makes dependencies explicit and avoids name conflicts from wildcard imports.

P19 BankAccount — deposit() and withdraw() 2024 Paper Q V · 7 marks

P22 Button Click Event — ActionListener 2023 Paper Q XII · 7 marks
To handle a button click: (1) implement ActionListener, (2) call button.addActionListener(this), (3) override actionPerformed(). e.getSource() identifies which component fired the event.
implements ActionListener addActionListener(this) actionPerformed() is the callback e.getSource() identifies source
Java (Swing)
import javax.swing.*; import java.awt.*; import java.awt.event.*; public class ButtonExample extends JFrame implements ActionListener { private JButton button; public ButtonExample() { setTitle("Button Event Demo"); setSize(250, 150); setLayout(new FlowLayout()); button = new JButton("Click Me"); add(button); button.addActionListener(this); // register listener setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); setVisible(true); } // called automatically when button is clicked public void actionPerformed(ActionEvent e) { if (e.getSource() == button) { JOptionPane.showMessageDialog(this, "Button was clicked!"); } } public static void main(String[] args) { ButtonExample aob = new ButtonExample(); } }

P23 JDBC Connection — 6 Steps 2023 Paper · Teacher's list · 7 marks
JDBC (Java Database Connectivity) allows Java to communicate with databases. There are exactly 6 steps to connect to a database: load driver → connect → create statement → execute → process results → close.
1. Load Driver → Class.forName("com.mysql.jdbc.Driver") 2. Get Connection → DriverManager.getConnection(url, user, pass) 3. Create Stmt → conn.createStatement() 4. Execute SQL → stmt.executeQuery("SELECT ...") 5. Process Result → while(rs.next()) { rs.getString("col") } 6. Close → conn.close()
Java (JDBC)
import java.sql.*; public class JDBCExample { public static void main(String[] args) { try { // Step 1: Load the JDBC driver Class.forName("com.mysql.jdbc.Driver"); // Step 2: Establish the connection Connection conn = DriverManager.getConnection( "jdbc:mysql://localhost/mydatabase", "username", "password"); // Step 3: Create a Statement Statement stmt = conn.createStatement(); // Step 4: Execute SQL query ResultSet rs = stmt.executeQuery( "SELECT * FROM mytable"); // Step 5: Process the ResultSet while (rs.next()) { int id = rs.getInt("id"); String name = rs.getString("name"); System.out.println("ID: " + id + ", Name: " + name); } // Step 6: Close the connection conn.close(); } catch (Exception e) { e.printStackTrace(); } } }
executeQuery() → for SELECT (returns ResultSet). executeUpdate() → for INSERT / UPDATE / DELETE (returns int rowcount).
P24 SQL Commands — CREATE, ALTER, DROP 2023 Paper Q XIII · 7 marks
These are DDL (Data Definition Language) commands — they define or modify the database structure. CREATE builds a new table; ALTER changes its structure; DROP permanently deletes the table and all its data.
CREATE → make new table ALTER → modify structure DROP → delete table permanently part of DDL
SQL
-- 1. CREATE: create a new table CREATE TABLE EMPLOYEE ( Name VARCHAR(20), Email VARCHAR(100), DOB DATE ); -- 2. ALTER: modify existing table structure -- Add a new column ALTER TABLE Customers ADD Email VARCHAR(255); -- Delete a column ALTER TABLE Customers DROP COLUMN Email; -- Rename a column ALTER TABLE Customers RENAME COLUMN Email TO Mail; -- Modify datatype of a column ALTER TABLE Customers MODIFY COLUMN Email VARCHAR(50); -- 3. DROP: permanently delete the table (structure + all data) DROP TABLE Employee;
DROP vs DELETE: DROP TABLE removes the whole table and its structure. DELETE FROM table removes rows but keeps the table structure.
