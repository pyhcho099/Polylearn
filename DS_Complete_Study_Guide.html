📦 Stack — Operations (Push, Pop, Traversal) 7 marks ▼
Stack is a linear data structure where insertion and deletion are done at only ONE end called the TOP. It follows LIFO (Last In First Out) principle.

Key Terms

  • TOP = pointer to topmost element. Initially top = −1 (empty stack)
  • MAXSIZE = maximum capacity of stack
  • Stack Overflow = trying to push when stack is full (top == MAXSIZE−1)
  • Stack Underflow = trying to pop from empty stack (top == −1)
  • Push = Insertion | Pop = Deletion

Push Algorithm

1If top == MAXSIZE−1 → print "Stack Overflow", stop
2Increment: top = top + 1
3S[top] = value (insert element)
4Stop

Pop Algorithm

1If top == −1 → print "Stack Underflow", stop
2val = S[top] (save top element)
3Decrement: top = top − 1
4Return/print val, Stop

Visual: Stack with A,B,C,D pushed

D← TOP
C
B
A
▔▔▔▔▔▔▔ Bottom
D was inserted last → D will be deleted first (LIFO)

After Pop → D removed, C becomes top

Applications of Stack

  • Infix to Postfix conversion
  • Evaluation of Postfix expressions
  • Recursion implementation
  • Parenthesis checker
  • Reversing a string/list

🔄 Infix → Postfix Conversion (Most Important!) 7 marks ▼
Infix: Operator between operands → A+B
Postfix: Operator after operands → AB+
Prefix: Operator before operands → +AB

Operator Priority (for conversion)

OperatorPriority

Algorithm: Infix to Postfix

1Add closing bracket ) at end of infix expression
2Push open bracket ( onto stack
3Scan each character left to right:
aIf ( → push onto stack
bIf operand (A–Z, 0–9) → add to postfix output
cIf ) → pop and output until ( found, then discard (
dIf operator → pop operators with higher or equal priority from stack to output, then push this operator
4Stack must be empty at end. If not → invalid expression

🚶 Queue — Operations (Insertion, Deletion) 7 marks ▼
Queue is a linear data structure where insertion is at REAR and deletion is at FRONT. It follows FIFO (First In First Out) principle.
Initially: front = rear = −1 (empty queue)

Key Terms

  • Enqueue = Insertion at rear
  • Dequeue = Deletion from front
  • Queue Overflow = Insert when rear == MAXSIZE−1
  • Queue Underflow = Delete when front == −1

Visual: Queue [10, 20, 30]

10
FRONT
20
30
REAR

Insertion Algorithm

1If rear == MAXSIZE−1 → "Queue Overflow", stop
2rear = rear + 1; Q[rear] = value
3If front == −1 → front = front + 1 (first element)
4Stop

Deletion Algorithm

1If front == −1 → "Queue empty", stop
2val = Q[front]; print val
3If front == rear → front = rear = −1 (last element deleted)
4Else front = front + 1
5Stop
⭐ Exam: In a queue, deletion is done from the _____ end

Answer: FRONT

🔁 Circular Queue (Very Important!) 7 marks ▼
Problem with Simple Queue: Even if positions at the front are empty, if rear == MAXSIZE−1, no insertion is allowed. This wastes memory!
Circular Queue: After rear reaches MAXSIZE−1, it wraps around to 0. The queue is full only when (rear+1) % MAXSIZE == front

Formula — Key Equations

Increment Rear
rear = (rear + 1) % MAXSIZE
Increment Front
front = (front + 1) % MAXSIZE
Queue Full Check
(rear+1) % MAXSIZE == front
Queue Empty
front == −1

Circular Queue Insertion

1If (rear+1)%MAXSIZE == front → "Queue Overflow", stop
2rear = (rear+1) % MAXSIZE; Q[rear] = value
3If front == −1 → front = 0
4Stop

Circular Queue Deletion

1If front == −1 → "Queue empty", stop
2Print Q[front]
3If front == rear → front = rear = −1 (reset, queue now empty)
4Else front = (front+1) % MAXSIZE
5Stop
⭐ Exam Q: How is circular queue better than linear queue?

Answer: In linear queue, rear=MAXSIZE−1 means full even if front positions are empty (wasted space). Circular queue reuses freed front positions by wrapping rear to index 0. Queue only full when (rear+1)%N == front.

↔️ Dequeue (Double Ended Queue) 3 marks ▼
Dequeue (Double Ended Queue) — Insertion and deletion can be done from BOTH ends (front and rear).

5 Operations

  1. Insertion at rear end (same as normal queue)
  2. Insertion at front end (decrement front by 1)
  3. Deletion from front end (same as normal queue)
  4. Deletion from rear end (decrement rear by 1)
  5. Traversal
Insertion at Front: If front == 0 → overflow (can't go left). Else front−−; Q[front] = val

Deletion from Rear: If single element → reset. Else rear−−

⭐ Priority Queue 3–7 marks ▼
Priority Queue: Each element has a priority value. Element with highest priority is deleted first. Elements are stored in order of priority so highest priority is always at front.

How Insertion Works

Find the correct position based on priority and insert there, shifting other elements right.

Example

Index01234
Value1020503040
Priority15678

Element with priority 1 is at front → deleted first. Deletion same as normal queue (remove from front).

📋 Linked List — Basics & Definition 1–3 marks ▼
Linked List is a dynamic data structure where elements (called nodes) are stored at any memory location and connected via pointers. Elements can only be accessed sequentially.

Node Structure

struct node {
    int data;           // stores the value
    struct node *next;  // pointer to next node
};

Array vs Linked List

ArrayLinked List
Fixed size (static)Dynamic size
Consecutive memoryRandom memory locations
Random access (index)Sequential access only
No pointer neededExtra pointer per node
Insertion/deletion slowInsertion/deletion fast

Types of Linked List

  • Singly Linked List — Each node has one pointer (to next)
  • Circular Linked List — Last node points back to first
  • Doubly Linked List — Each node has two pointers (prev and next)

➡️ Singly Linked List — All Operations 7 marks ▼

START (or HEAD) pointer stores address of first node. Last node's next = NULL.

START→[1|•]→[2|•]→[3|•]→[4|•]→[5|NULL]

1. Insertion at Beginning

1Allocate new node: ptr = malloc(sizeof(struct node))
2ptr→data = value
3ptr→next = start (point to old first node)
4start = ptr (new node is now first)

2. Insertion at End

1Allocate new node; ptr→data = value; ptr→next = NULL
2Traverse to last node (where temp→next == NULL)
3temp→next = ptr

3. Insertion After Given Node

1Allocate new node
2Traverse to find the given node (pretemp)
3ptr→next = pretemp→next
4pretemp→next = ptr

4. Deletion of First Node

1If start == NULL → empty list
2ptr = start
3start = start→next
4free(ptr)

5. Deletion of Last Node

1If start == NULL → empty list
2If start→next == NULL → only 1 node, free it, start = NULL
3Traverse with pretemp and temp until temp→next == NULL
4pretemp→next = NULL; free(temp)

6. Deletion of Specific Node

1If empty → error. If first node matches → delete first
2Traverse with pretemp and temp until temp→data == val or temp == NULL
3If temp == NULL → not found
4pretemp→next = temp→next; free(temp)

📦 Linked Stack & Linked Queue 7 marks ▼

Linked Stack

Stack implemented using linked list. The START pointer acts as TOP. All insertions and deletions happen at the node pointed by top.

Push (Insert at beginning of list)

1Allocate node: ptr→data = val; ptr→next = top
2top = ptr

Pop (Delete from beginning)

1If top == NULL → "Stack Underflow"
2ptr = top; top = top→next; free(ptr)
No overflow in Linked Stack — memory is dynamic! Only underflow is possible (when top == NULL).

Linked Queue

Queue using linked list. START = FRONT, last node = REAR. Insert at rear, delete from front.

Enqueue (Insert at rear)

1Allocate node: ptr→data = val; ptr→next = NULL
2If front == NULL → front = rear = ptr
3Else: rear→next = ptr; rear = ptr

Dequeue (Delete from front)

1If front == NULL → "Queue empty"
2ptr = front; print ptr→data
3If front == rear → front = rear = NULL
4Else front = front→next; free(ptr)

🔍 Binary Search Tree (BST) — Creation & Operations 7 marks ▼
BST Rule: For each node,
• Left subtree has nodes with values < parent
• Right subtree has nodes with values >= parent

3100 > 50 → right of 50
430 < 50 → go left (20); 30 > 20 → right of 20
5200 > 50 → go right (100); 200 > 100 → right of 100
610 < 50 → go left (20); 10 < 20 → left of 20

          50
        /    \
      20      100
     /  \        \
   10   30       200
                 /
               150

BST Search Algorithm

1Start at root
2If value == root → FOUND
3If value < root → search left subtree
4If value > root → search right subtree
5If NULL reached → NOT FOUND

BST Deletion — 3 Cases

Case 1: Leaf Node
Simply remove the node. Set parent's pointer to NULL.
Case 2: One Child
Replace node with its only child. Parent points to that child.
Case 3: Two Children
Replace node's value with its inorder successor (smallest node in right subtree) or inorder predecessor (largest in left subtree). Then delete that successor/predecessor node.

Binary tree used to store algebraic expressions. Operands are at leaf nodes. Operators are at internal nodes.

  • Postorder traversal → Postfix expression

Threaded Binary Tree

TBT reuses NULL pointers:
• If left child is NULL → point to inorder predecessor
• If right child is NULL → point to inorder successor
These special pointers are called threads (shown as dotted lines).
In a binary tree with n nodes, there are n+1 NULL pointers. TBT uses these efficiently.

🔎 DFS — Depth First Search (Very Important!) 7 marks ▼
DFS uses a STACK. Goes as deep as possible along each branch before backtracking.

DFS Algorithm

1Define a stack of size = number of vertices
2Select any starting vertex, visit it and push onto stack
3Visit any ONE non-visited adjacent vertex of top of stack, push it
4Repeat step 3 until no unvisited adjacent vertex found
5Pop one vertex from stack
6Repeat steps 3,4,5 until stack empty

BFS — Breadth First Search

BFS uses a QUEUE. Visits all neighbors at current depth before going deeper.
1Define a queue. Select start vertex, visit it, insert into queue
2Visit ALL non-visited adjacent vertices of front element, insert them
3Delete the front element from queue
4Repeat steps 2,3 until queue is empty

DFS vs BFS Comparison

DFSBFS
Uses StackUses Queue
Goes deep firstExplores neighbors first
May not find shortest pathFinds shortest path (unweighted)
Less memory (recursion)More memory (all neighbors stored)

Example: BFS on graph A-B-C-D-E-F-G

Graph: A connected to B,D,E; B connected to C; E connected to F; C connected to G

BFS from A:
Visit A → queue:[A]
Visit B,D,E (neighbors of A) → queue:[B,D,E], dequeue A
Visit C (neighbor of B) → queue:[D,E,C], dequeue B
Visit F (neighbor of E) → queue:[C,F], dequeue D,E
Visit G (neighbor of C) → queue:[F,G], dequeue C
Dequeue F,G
BFS order: A B D E C F G

🗺️ Warshall's Shortest Path Algorithm 7 marks ▼
Floyd-Warshall Algorithm finds the shortest paths between ALL pairs of vertices in a weighted graph.

Adjacency matrix entry: A[i][j] = weight if edge exists, else = ∞ (infinity)

Key Formula

A[i][j] = min(A[i][j], A[i][k] + A[k][j])

For each intermediate vertex k, check if going through k gives a shorter path from i to j.

Algorithm Steps

1Initialize matrix A0 = cost adjacency matrix (∞ where no edge)
2For k = 1 to n:
  For i = 1 to n:
    For j = 1 to n:
      A[i][j] = min(A[i][j], A[i][k] + A[k][j])
3Final matrix An = shortest path matrix

Q: Explain Dequeue (3 marks)

Dequeue = Double Ended Queue. Insertion and deletion possible at BOTH ends (front and rear).
5 Operations: (1) Insert at rear (2) Insert at front (3) Delete from front (4) Delete from rear (5) Traversal.
Insert at front: decrement front by 1, Q[front]=val. Delete from rear: decrement rear by 1.

Q: Infix, Prefix, Postfix with examples (3 marks)

Infix: Operator between operands. Example: A+B. Human readable.
Prefix: Operator before operands. Example: +AB. Also called Polish notation.
Postfix: Operator after operands. Example: AB+. Also called Reverse Polish. Parenthesis-free. Used by computers.
More example: (A+B)*C → AB+C* (postfix), *+ABC (prefix)

Q: Applications of Stack (3 marks)

1. Infix to Postfix conversion — uses stack to handle operator precedence
2. Evaluation of postfix expressions — stack stores operands
3. Recursion — function call stack
4. Parenthesis checker — check balanced brackets

Q: Types of Linked List (3 marks)

1. Singly Linked List: Each node has one pointer (to next node). Traversal in one direction only.
2. Circular Linked List: Last node points to first node. No NULL at end.
3. Doubly Linked List: Each node has two pointers (prev and next). Can traverse both ways. More memory used.

Q: Write a note about perfect binary tree (3 marks)

Perfect Binary Tree: All leaf nodes are at the SAME depth/level. All non-leaf (internal) nodes have exactly two children.
Number of nodes at level h = 2^h. Total nodes = 2^(h+1) − 1 for tree of height h.
Example: Tree with root and 2 children, each having 2 children = 7 nodes total.

Q: Weighted Graph and Directed Graph (3 marks)

Weighted Graph: Every edge has a numerical weight (cost, distance). Adjacency matrix called cost adjacency matrix. A[i][j] = weight if edge exists, ∞ if no edge.

Directed Graph (Digraph): Edges have direction shown by arrows. Edge (A,B) means path from A→B only. Has in-degree and out-degree for each vertex.

Q: Deletion from beginning of singly linked list (3 marks)

1. If start == NULL → list is empty, deletion not possible
2. ptr = start
3. start = start→next (start now points to second node)
4. free(ptr) (deallocate memory of deleted node)
After deletion, start points to what was the second node.
