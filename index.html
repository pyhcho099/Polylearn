P17 marks
Explain the features of Object Oriented Programming (OOP).
All 3 papers · Teacher's list · Part C

OOP has 8 main features:

  1. Object — Real-world entities with their own properties (data) and behavior (methods). Has physical existence. Eg: person, bank, company.
  2. Class — A blueprint or prototype from which objects are created. A generalized description of an object.
  3. Encapsulation — Wrapping up of data (variables) and functions (methods) into a single unit called a class. Keeps data safe from outside interference.
  4. Abstraction — Displaying only essential information and hiding the background details or implementation from the user.
  5. Polymorphism — The ability of a message/method to be displayed in more than one form. Eg: method overloading and overriding.
  6. Inheritance — The capability of a class to derive properties and characteristics from another class. Promotes code reusability.
  7. Dynamic Binding — The code associated with a given procedure call is not known until the time of the call at runtime.
  8. Message Passing — A request for execution of a procedure. Involves specifying the object name, method name, and information to be sent.
Scoring tip: List all 8 (1 mark) + explain any 4 at 1.5 marks each = 7 marks total.

Explain Class and Object in Java with examples.
Teacher's list · Core OOP Concept

Class (3 marks)

  • A class is a blueprint or template from which objects are created.
  • It defines the attributes (variables) and behavior (methods) that the objects of the class will have.
  • A class does not occupy memory by itself — memory is allocated only when an object is created.
  • Syntax: class ClassName { attributes; methods; }
class Student { String name; // attribute int age; // attribute void display() { // method System.out.println("Name: " + name + ", Age: " + age); } }

Object (3 marks)

  • An object is a real-world entity that has a state (attributes) and behavior (methods).
  • An object is an instance of a class — created using the new keyword.
  • Each object has its own copy of the class attributes.
  • Syntax: ClassName objectName = new ClassName();
public class Main { public static void main(String[] args) { Student s1 = new Student(); // object creation s1.name = "Alice"; s1.age = 20; s1.display(); // Output: Name: Alice, Age: 20 Student s2 = new Student(); // another object s2.name = "Bob"; s2.age = 22; s2.display(); // Output: Name: Bob, Age: 22 } }

What is Inheritance? Explain the different types of inheritance in Java.
2023 & 2024 Papers · Teacher's list · Part C

Definition (2 marks)

Inheritance in Java is a mechanism in which one class (child/subclass) acquires all the properties and behaviors of another class (parent/superclass). It promotes code reusability. The keyword used is extends.

Types of Inheritance (5 marks — 1 each)

  1. Single Inheritance — A class inherits from only one superclass.
    class Animal { } class Dog extends Animal { }
  2. Multiple Inheritance — A class inherits from more than one superclass. Not directly supported in Java — achieved through interfaces.
  3. Hierarchical Inheritance — One superclass is inherited by multiple subclasses.
    class Animal { } class Dog extends Animal { } class Cat extends Animal { }
  4. Multi-Level Inheritance — A class is derived from a class which is also derived from another class (chain of inheritance).
    class Animal { } class Dog extends Animal { } class Puppy extends Dog { }
  5. Hybrid Inheritance — Combination of more than two types of inheritance. Implemented in Java using interfaces.

Explain the steps for creating a JDBC connection in Java.
2023 Paper · Teacher's list (Exam 23/03) · Part C

There are 6 steps for a Java program to access a database using JDBC:

  1. Load the JDBC Driver — Load the appropriate driver class for the database.
    Class.forName("com.mysql.jdbc.Driver");
  2. Establish a Connection — Use DriverManager.getConnection() with URL, username, and password.
    Connection conn = DriverManager.getConnection( "jdbc:mysql://localhost/mydatabase", "username", "password");
  3. Create a Statement — Create a Statement object to execute SQL.
    Statement stmt = conn.createStatement();
  4. Execute SQL Statements — Use executeQuery() for SELECT, executeUpdate() for INSERT/UPDATE/DELETE.
    ResultSet rs = stmt.executeQuery("SELECT * FROM mytable");
  5. Process the Results — Traverse the ResultSet using next() and retrieve data.
    while (rs.next()) { int id = rs.getInt("id"); String name = rs.getString("name"); System.out.println("ID: " + id + ", Name: " + name); }
  6. Close the Connection — Always close to free resources.
    conn.close();

Teacher's list · 2023 Paper (Q VIII option)

1. Final Variable (2 marks)

A variable declared with the final keyword whose value cannot be changed once assigned. It acts as a constant.

A method declared with final that cannot be overridden by any subclass. Used to prevent changing critical behavior.

A class declared with final that cannot be extended/inherited by any other class. All methods in a final class are implicitly final.

final public class MyFinalClass { void show() { System.out.println("Final class"); } } // class SubClass extends MyFinalClass { } // ERROR

Example of a real final class in Java: java.lang.String is a final class.

P17 marks
Define abstract class and abstract method in Java. Give an example.
Teacher's list · Model QP (Q VIII) · Part C

Abstract Class (3 marks)

  • A class declared with the abstract keyword.
  • Cannot be instantiated (cannot create objects directly).
  • Can have both abstract methods (no body) and concrete methods (with body).
  • Must be extended by a subclass that implements all abstract methods.
  • If a class does not implement all abstract methods, it must also be declared abstract.

Abstract Method (2 marks)

  • A method declared with abstract keyword and no method body.
  • Ends with a semicolon instead of a body block.
  • Syntax: abstract returnType methodName();
  • Forces subclasses to provide their own implementation.

Example (2 marks)

abstract class Animal { String name; abstract void makeSound(); // abstract — no body void breathe() { // concrete method System.out.println("Breathing..."); } } class Dog extends Animal { void makeSound() { // must implement System.out.println("Bark!"); } } public class Main { public static void main(String[] args) { Dog d = new Dog(); d.makeSound(); // Output: Bark! d.breathe(); // Output: Breathing... } }

Explain Interface in Java with an example. How is it used for multiple inheritance?
Teacher's list · 2023 Paper (Q X) · 2024 Paper (Q IX) · Part C

Definition (2 marks)

An interface is a reference type in Java, similar to a class. It contains only abstract method signatures and constants. Declared using interface keyword; implemented using implements keyword.

Features (2 marks)

  • All methods are public and abstract by default.
  • All variables are public, static, and final by default.
  • A class can implement multiple interfaces — achieving multiple inheritance.
  • An interface can extend multiple interfaces.
  • Cannot be instantiated.

Example — Multiple Inheritance via Interface (3 marks)

interface Animal { void makeSound(); void eat(); } interface Pet { void play(); } class Dog implements Animal, Pet { // multiple inheritance public void makeSound() { System.out.println("Woof!"); } public void eat() { System.out.println("Eating dog food"); } public void play() { System.out.println("Playing fetch"); } } public class Main { public static void main(String[] args) { Dog d = new Dog(); d.makeSound(); // Woof! d.eat(); // Eating dog food d.play(); // Playing fetch } }

Java supports static members — global methods and variables that can be accessed without creating objects of a class. They are prefixed with the keyword static. (3 marks)

Static Variables (3 marks)

  • Memory is allocated only once when the class is loaded.
  • Used to represent class-level properties.

  • Cannot use this or super keywords.

Definition (2 marks)

The super keyword in Java is a reference variable used to refer to the immediate parent class object. It is used when parent and child class have same attribute names.

Uses of super keyword (4 marks — 1 each)

  1. Refer to the immediate parent class instance variable: super.variableName
  2. Invoke the immediate parent class method: super.methodName()
  3. Invoke the immediate parent class constructor: super() — must be first statement
  4. Access parent class data when parent and child have same attribute name.

Explain the features of Java Swing.
Teacher's list (Exam 16/03) · Part C
  1. Part of JFC (Java Foundation Classes) — Swing is included in the Java Foundation Classes library.
  2. Built on AWT — Swing is built on top of AWT (Abstract Window Toolkit) but provides richer components.

  8. Double buffering — Provides smoother rendering by drawing off-screen before displaying.
  9. Package: javax.swing — All Swing components are in the javax.swing package.

Explain the different layouts in Java Swing (Layout Managers).
Teacher's list (Exam 16/03) · Part C

A Layout Manager controls the position and size of components within a container.

  1. FlowLayout — Components are placed from left to right, then wrap to the next line. Default for JPanel.
    frame.setLayout(new FlowLayout());
  2. BorderLayout — Divides the container into 5 regions: North, South, East, West, and Center. Default for JFrame.
    frame.setLayout(new BorderLayout()); frame.add(btn, BorderLayout.NORTH);
  3. GridLayout — Arranges components in a grid of equal-sized rows and columns.
    frame.setLayout(new GridLayout(3, 2)); // 3 rows, 2 cols
  4. GridBagLayout — The most flexible layout. Components can span multiple cells in a grid. Complex to use but very powerful.
  5. CardLayout — Multiple components share the same display space; only one is visible at a time (like cards in a deck). Used for tabbed interfaces.
  6. BoxLayout — Arranges components in a single row or single column.
    frame.setLayout(new BoxLayout(frame, BoxLayout.Y_AXIS));

P27 marks
Explain DDL, DML, and DCL commands with examples.
Teacher's list (Exam 23/03) · Part C

DDL — Data Definition Language (3 marks)

Commands that define the structure of the database. Changes are auto-committed.

Commands: CREATE, ALTER, DROP, TRUNCATE, RENAME

CREATE TABLE Student (id INT, name VARCHAR(50), age INT); ALTER TABLE Student ADD email VARCHAR(100); DROP TABLE Student; TRUNCATE TABLE Student; -- deletes all rows, keeps structure

DML — Data Manipulation Language (3 marks)

Commands that manipulate the data in the database tables. Changes can be rolled back.

Commands: INSERT, UPDATE, DELETE, SELECT

INSERT INTO Student VALUES (1, 'John', 20); UPDATE Student SET age = 21 WHERE id = 1; DELETE FROM Student WHERE id = 1; SELECT * FROM Student WHERE age > 18;

DCL — Data Control Language (1 mark)

Commands that control access (permissions) to database objects.

Commands: GRANT, REVOKE

GRANT SELECT ON Student TO user1; REVOKE SELECT ON Student FROM user1;

Explain the different interfaces in JDBC.
Teacher's list (Exam 23/03) · Part C
  1. Connection — Represents a connection to a specific database. Used to create Statement objects.
    Connection conn = DriverManager.getConnection(url, user, pwd);
  2. Statement — Used to execute simple SQL queries. Does not accept parameters.
    Statement stmt = conn.createStatement();
  3. PreparedStatement — A precompiled SQL statement that can accept input parameters. More efficient and secure (prevents SQL injection).
    PreparedStatement ps = conn.prepareStatement("SELECT * FROM student WHERE id=?"); ps.setInt(1, studentId);
  4. CallableStatement — Used to execute stored procedures in the database.
    CallableStatement cs = conn.prepareCall("{call myProcedure(?)}");
  5. ResultSet — Represents the result of a SELECT query. Provides methods like next(), getString(), getInt() to traverse and read data.
    ResultSet rs = stmt.executeQuery("SELECT * FROM student");
  6. DatabaseMetaData — Provides information about the database (name, version, tables, etc.).
  7. ResultSetMetaData — Provides information about the columns in a ResultSet (column names, types, count).

Write the significance of Java Virtual Machine (JVM).
2023 Paper (Q II.1) · Answer Key provided
  • JVM (Java Virtual Machine) is a program that provides the runtime environment to execute Java programs.
  • JVM is a virtual machine that resides in the real machine.
  • The machine language for JVM is bytecode. The Java compiler generates bytecode rather than machine code for each hardware type.
  • JVM executes the bytecode generated by the compiler and produces output.
  • JVM is the reason Java is platform independent — the same bytecode runs on any machine that has a JVM.

What is meant by bytecode?
2024 Paper (Q II.1) · Part B
  • Bytecode is the intermediate code generated by the Java compiler (javac).
  • It is not machine code — it cannot be run directly on any hardware.
  • Bytecode is stored in .class files.
  • Bytecode is executed by the JVM (Java Virtual Machine), which translates it to the appropriate machine code for the current platform.
  • This makes Java programs platform independent — write once, run anywhere.

What is meant by relational database?
2024 Paper (Q II.8) · Part B
  • A relational database is a type of database that stores and organizes data in tables (rows and columns).
  • Each table represents a relation; rows are records, columns are attributes.
  • Tables are linked to each other using keys (primary key and foreign key).
