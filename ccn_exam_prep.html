Describe Random Access Protocol – ALOHA (Pure and Slotted)
▾
Answer

ALOHA is a random access protocol where stations transmit data without checking the medium first. There are two versions:

1. Pure ALOHA: Stations transmit whenever they have data. If a collision occurs, they wait a random time and retransmit.

  • Vulnerable period = 2 × Tfr (frame transmission time)
  • Maximum throughput = 18.4% at load G = 0.5
PURE ALOHA THROUGHPUT FORMULA
S = G × e−2G

2. Slotted ALOHA: Time is divided into slots equal to frame transmission time. Stations may only transmit at the start of a slot.

  • Vulnerable period = Tfr (half that of Pure ALOHA)
  • Maximum throughput = 37% at load G = 1
SLOTTED ALOHA THROUGHPUT FORMULA
S = G × e−G
Pure ALOHA vs Slotted ALOHA — Timeline

Transmission TimeAny timeOnly at slot boundaries
Vulnerable Time2 × TfrTfr
Max Throughput18.4% at G=0.537% at G=1
EfficiencyLowerTwice that of Pure ALOHA
ComplexitySimplerRequires time synchronization

Describe CSMA/CD Protocol with Diagram (Ethernet)
▾
Answer

CSMA/CD (Carrier Sense Multiple Access with Collision Detection) is a MAC protocol used in wired Ethernet (IEEE 802.3). It detects collisions after they occur and recovers from them.

  • Carrier Sense: Listen before transmitting.
  • Multiple Access: Multiple stations share the medium.
  • Collision Detection: Monitor while transmitting; detect voltage spikes.
  • Jam Signal: 32-bit signal sent to alert all stations of collision.
  • Binary Exponential Backoff: Wait time = random(0, 2ᵏ−1) × slot time, where k = retry count.
  • Minimum Frame Size: 64 bytes (ensures collision is detected during transmission).
  • Standard: IEEE 802.3 (Ethernet)

▾
Answer

SMTP (Simple Mail Transfer Protocol) is a push protocol used for sending emails from client to server and between servers. It operates on TCP Port 25 and uses 7-bit ASCII text-based commands.

Email Architecture — Mail Flow Diagram

SMTP Commands: HELO, MAIL FROM, RCPT TO, DATA, QUIT.

MeritsDemerits
Simple, text-based, widely supportedOnly supports 7-bit ASCII natively
Reliable delivery with acknowledgmentNo built-in encryption (needs TLS/SSL)
Persistent connections for efficiencyVulnerable to spam and email spoofing

Describe Distance Vector Routing with Advantages and Disadvantages
▾
Answer

Distance Vector Routing is a dynamic routing algorithm where each router maintains a table of distances (costs) to every destination. Routers share their complete table only with direct neighbors periodically.

Algorithm (Bellman-Ford):

BELLMAN-FORD EQUATION
Dx(y) = minv { C(x,v) + Dv(y) }

Where C(x,v) is the cost from x to neighbor v, and Dv(y) is v's known distance to destination y.

Distance Vector — Router Sharing
AdvantagesDisadvantages
Simple to implementCount-to-infinity problem
Low memory usageSlow convergence
Good for small networksRouting loops possible
Low processing overheadBandwidth wasted sharing full tables

Solutions to Count-to-Infinity: Split Horizon (don't advertise route back to source), Poison Reverse (advertise with infinite metric ∞), Hold-down timers.

Describe Stop-and-Wait Protocol with Diagram
▾
Answer

Stop-and-Wait is the simplest flow control protocol. The sender transmits one frame and waits for an acknowledgment (ACK) before sending the next. If no ACK arrives within a timeout, the frame is retransmitted.

Stop-and-Wait — Timeline Diagram
  • Advantage: Simple, easy to implement.
  • Disadvantage: Low efficiency — link is idle while waiting for ACK.
  • Efficiency η = Tfr / (Tfr + 2×Tp) where Tp is propagation delay.

Define Framing and Various Methods Used in Framing
▾
Answer

Framing is the Data Link Layer function of encapsulating a network layer datagram into a frame by adding a header and trailer, marking the start and end of each frame so the receiver can identify boundaries.

MethodDescription
Character CountHeader field specifies number of characters in the frame.
Flag Bytes with Byte StuffingSpecial flag byte (01111110) marks start/end. Escape byte inserted before flag in data.
Starting/Ending Flags with Bit StuffingFlag pattern (01111110) marks boundaries. A 0-bit inserted after five consecutive 1s in data.
Physical Layer Coding ViolationsUses invalid signal patterns (not used in data) to indicate frame boundaries.

List the SSH Components and Describe SSH Protocol
▾
Answer

SSH (Secure Shell) is a cryptographic network protocol for secure remote login and command execution over an unsecured network. It uses TCP Port 22.

SSH ComponentFunction
SSH-TRANS (Transport Layer Protocol)Provides server authentication, confidentiality, and integrity. Creates the secure encrypted tunnel.
SSH-AUTH (User Authentication Protocol)Authenticates the client to the server using passwords, public keys, or host-based methods.
SSH-CONN (Connection Protocol)Multiplexes the secure tunnel into multiple logical channels (terminal sessions, file transfers, etc.).
SSH ApplicationsSFTP (Secure File Transfer), SCP (Secure Copy), Remote Login (replacing Telnet).
